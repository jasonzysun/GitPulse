# Product

## Register

product

## Users

GitPulse is used by developers who need to turn local Git activity across multiple repositories into daily, weekly, custom-period, or monthly work reports. They are usually preparing status updates, weekly summaries, project reviews, or performance evidence under time pressure.

## Product Purpose

GitPulse reduces the repetitive work of searching `git log`, grouping commits, and rewriting scattered changes into a usable report. Success means a user can select a workspace, choose a report period, generate a trustworthy Markdown report, and copy or export it without sending source activity to a remote service.

## Brand Personality

Quiet, capable, trustworthy. The product should feel like a focused local workbench: efficient, clear, and respectful of sensitive developer data.

## Anti-references

Avoid marketing-page layouts, decorative dashboard cards, hidden controls, playful novelty UI, and visual treatments that make routine reporting feel heavier than the task itself.

## Design Principles

- Keep report generation controls visible where the report is generated.
- Make local-first privacy and traceability feel dependable through plain, concrete UI.
- Favor predictable desktop tool patterns over surprising custom interactions.
- Preserve density without hiding primary actions.
- Show progress and selected scope during long-running local Git operations.

## Accessibility & Inclusion

Use clear labels, visible focus states, sufficient contrast in light and dark themes, and controls that remain understandable without relying on hover-only discovery.
