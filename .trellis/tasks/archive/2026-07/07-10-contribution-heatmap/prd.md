# 工作热力图（Contribution Heatmap）

## Goal

提供类似 GitHub contribution graph 的热力图组件，聚合所有本地仓库的 commit 数据，按日期展示开发活跃度。

## Requirements

### 后端（Rust）

- 新增 Tauri command：接收作者、日期范围（默认最近 52 周），返回 `Vec<{date: String, count: u32}>` 格式的每日 commit 计数
- 复用已有仓库索引和 commit 提取管线，不重复扫描
- 跨仓库去重：同一 commit（同 hash）出现在多个仓库时只计一次

### 前端

- 热力图组件：7 行（周一到周日）× N 列（周数）的格子矩阵
- 颜色梯度：无提交（灰色/背景色）→ 少量 → 中等 → 活跃 → 高活跃（4-5 级）
- 配色需适配明暗两种主题
- 鼠标悬浮显示具体日期和提交数
- 显示月份标签、星期标签
- 底部显示总结：总提交数、活跃天数、最长连续活跃天数、最活跃的一天

### 放置位置

- 工作台主界面，在报告生成区域上方或作为独立 tab/section
- 响应式布局，窄屏时可横向滚动

## Acceptance Criteria

- [ ] 热力图正确聚合所有已索引仓库的 commit 数据
- [ ] 52 周数据展示，格子颜色梯度正确
- [ ] 明暗主题均适配
- [ ] 悬浮提示显示日期和提交数
- [ ] 跨仓库同一 commit 不重复计数
- [ ] 底部统计摘要正确（总数、活跃天、最长连续、最活跃日）
- [ ] 性能：1000+ 仓库场景下渲染不卡顿
