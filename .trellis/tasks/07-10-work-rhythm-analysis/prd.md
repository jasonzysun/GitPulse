# 工作节奏分析

## Goal

分析开发者的工作时间模式，提供可视化的工作节奏报告，帮助个人了解自身习惯、团队管理者了解工作负载分布。

## Requirements

### 后端（Rust）

- 新增 Tauri command：接收作者、日期范围，返回结构化的时间分析数据
- 数据维度：
  - 每小时提交分布（0-23 时，每小时的 commit 计数）
  - 每日提交分布（周一到周日）
  - 工作时间 vs 非工作时间比例（工作时间定义：工作日 9:00-18:00，可配置）
  - "加班"提交比例（工作日 21:00 后 + 周末，阈值可配置）
- 本周 vs 上周对比数据

### 前端

- 时间分布图：24 小时 × 7 天的气泡图或热力矩阵，展示什么时间段最活跃
- 工作节奏卡片：
  - 最活跃时段（如"下午 2-4 点"）
  - 本周工作日加班提交占比
  - 周末提交占比
  - 本周 vs 上周提交数对比（带增减箭头）
- 适配明暗主题

### 放置位置

- 可以与热力图放在同一个"数据洞察"面板中

## Acceptance Criteria

- [ ] 24 小时提交分布数据正确
- [ ] 工作日/非工作日分类正确（考虑时区）
- [ ] 加班比例计算正确
- [ ] 本周 vs 上周对比数据正确
- [ ] 可视化组件明暗主题适配
- [ ] 工作时间定义可在设置中配置
