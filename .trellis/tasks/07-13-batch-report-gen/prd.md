# 支持批量生成工作报告

> GitHub Issue: https://github.com/GoldenZqqq/GitPulse/issues/1
> 提出者: jasonzysun

## Goal

允许用户一次性生成多份报告（如一个月的30份日报、4份周报），避免逐份手动操作。覆盖补历史记录、批量归档等真实场景。

## Background

当前 GitPulse 每次交互只能生成一份报告。用户如果需要补一个月的日报，需要手动切换日期30次。批量生成功能将复用现有的 commit 提取、报告渲染、格式导出基建，在外层加一个"日期拆分 + 循环生成 + 进度反馈"的编排层。

## Requirements

### Phase 1 (MVP)

- 用户选择一个时间范围（start_date ~ end_date）和拆分粒度（按天/按周/按月）
- 系统自动将时间范围拆分为 N 个子周期
- 对每个子周期调用现有报告生成流程，生成对应报告
- 支持选择导出格式（Markdown / DOCX / PDF），每份报告导出为独立文件
- 用户选择输出目录
- 文件命名规则：`{日期范围}-{报告类型}.{ext}`（如 `2026-07-01-日报.md`、`2026-W28-周报.docx`）
- 生成过程中显示进度（当前第 N/M 份）
- 生成完成后显示成功/失败汇总，允许打开输出目录
- 批量生成复用现有筛选条件（作者、仓库、分支）

### Phase 2 (Future)

- 按作者分组：为每位作者分别生成独立报告
- 按项目分组：为每个项目分别生成独立报告
- 自定义文件命名模板：`{date}-{author}-{project}-{type}.{ext}`
- 同时导出多种格式
- 可选 AI 润色（批量场景下需考虑 API 限流和费用）

## Constraints

- 批量生成期间不阻塞 UI，使用 Tauri event 推送进度
- 单次批量上限建议 365 份，防止意外操作
- 复用现有 `commit_pipeline`、`report`、`docx`、`pdf` 模块，不重复造轮子
- 文件写入失败不中断整个批量流程，记录失败项并最终汇总

## Acceptance Criteria

- [ ] 用户可在前端选择时间范围 + 拆分粒度 + 导出格式 + 输出目录
- [ ] 后端按拆分粒度将时间范围切分为子周期，逐份生成报告
- [ ] 每份报告内容与单独生成该周期报告的结果一致
- [ ] 生成过程中前端显示实时进度
- [ ] 生成完成后显示成功/失败汇总
- [ ] 单份失败不影响后续报告生成
- [ ] 文件命名包含日期和报告类型，无冲突
- [ ] 支持 Markdown、DOCX、PDF 三种格式
